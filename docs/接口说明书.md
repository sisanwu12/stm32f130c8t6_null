# dri_ll_gpio 接口说明书

## 1. 模块概述

`dri_ll_gpio` 是面向 `STM32F103` 的 GPIO 底层驱动模块，采用三层封装方式实现：

- 地址定义层：负责 GPIO 相关基地址、寄存器偏移和寄存器映射定义。
- 基础操作层：负责直接操作寄存器，提供引脚配置、原始输入输出读写等能力。
- 用户接口层：负责对外提供更易使用的初始化、单引脚控制、整口读写等接口。

当前模块文件：

- `driver/dri_ll_gpio.h`
- `driver/dri_ll_gpio.c`

---

## 2. 适用范围

- MCU 系列：`STM32F103`
- GPIO 端口：`GPIOA`、`GPIOB`、`GPIOC`、`GPIOD`、`GPIOE`
- 引脚范围：每个端口 `Pin0 ~ Pin15`
- 驱动风格：裸机寄存器驱动，不依赖官方 HAL/StdPeriph 库

---

## 3. 分层说明

### 3.1 地址定义层

地址定义层主要放在 `dri_ll_gpio.h` 中，用于描述 GPIO 的寄存器布局。

主要内容：

- 外设基地址定义
- GPIOA ~ GPIOE 起始地址与结束地址定义
- GPIO 寄存器偏移定义
- `dri_ll_gpio_reg_t` 寄存器映射结构体

常用宏示例：

| 宏名                     | 说明             |
| ------------------------ | ---------------- |
| `DRI_LL_GPIOA_BASE_ADDR` | GPIOA 起始地址   |
| `DRI_LL_GPIOB_BASE_ADDR` | GPIOB 起始地址   |
| `DRI_LL_GPIO_CRL_OFFSET` | `CRL` 寄存器偏移 |
| `DRI_LL_GPIO_CRH_OFFSET` | `CRH` 寄存器偏移 |
| `DRI_LL_GPIOA`           | GPIOA 寄存器入口 |

GPIO 寄存器映射如下：

| 成员   | 说明                |
| ------ | ------------------- |
| `CRL`  | 配置 `Pin0 ~ Pin7`  |
| `CRH`  | 配置 `Pin8 ~ Pin15` |
| `IDR`  | 输入数据寄存器      |
| `ODR`  | 输出数据寄存器      |
| `BSRR` | 原子置位/复位寄存器 |
| `BRR`  | 复位寄存器          |
| `LCKR` | 配置锁定寄存器      |

---

### 3.2 基础操作层

基础操作层面向“直接寄存器访问”，适合驱动开发者或后续扩展模块调用。

#### 端口与引脚类型

| 类型                 | 说明                                                            |
| -------------------- | --------------------------------------------------------------- |
| `dri_ll_gpio_port_t` | GPIO 端口枚举，范围为 `DRI_LL_GPIO_PORT_A ~ DRI_LL_GPIO_PORT_E` |
| `dri_ll_gpio_pin_t`  | GPIO 引脚枚举，范围为 `DRI_LL_GPIO_PIN_0 ~ DRI_LL_GPIO_PIN_15`  |

#### 基础操作接口

| 接口                                                     | 返回值               | 说明                                  |
| -------------------------------------------------------- | -------------------- | ------------------------------------- |
| `dri_ll_gpio_get_reg(port)`                              | `dri_ll_gpio_reg_t*` | 返回端口寄存器入口，非法端口返回 `NULL` |
| `dri_ll_gpio_get_pin_mask(pin)`                          | `uint16_t`           | 返回引脚位掩码，非法引脚返回 `0`        |
| `dri_ll_gpio_config_pin_raw(port, pin, cfg_bits)`        | `void`               | 直接写目标引脚的 4bit 配置字段          |
| `dri_ll_gpio_write_mask_raw(port, set_mask, reset_mask)` | `void`               | 通过 `BSRR` 原子置位/复位               |
| `dri_ll_gpio_read_input_raw(port)`                       | `uint16_t`           | 返回 `IDR[15:0]`，非法端口返回 `0`      |
| `dri_ll_gpio_read_output_raw(port)`                      | `uint16_t`           | 返回 `ODR[15:0]`，非法端口返回 `0`      |
| `dri_ll_gpio_write_output_raw(port, value)`              | `void`               | 直接写整个端口的 `ODR`                  |

#### `cfg_bits` 说明

`STM32F1` 的每个 GPIO 引脚在 `CRL/CRH` 中占用 4bit 配置字段：

- 高 2 位：`CNF[1:0]`
- 低 2 位：`MODE[1:0]`

本驱动直接使用 4bit 组合作为配置值，因此可以非常贴近参考手册。

### 3.3 用户接口层

用户接口层面向应用开发者，屏蔽了一部分寄存器细节。

#### 电平类型

| 类型                  | 取值                     | 说明   |
| --------------------- | ------------------------ | ------ |
| `dri_ll_gpio_level_t` | `GPIO_LEVEL_LOW`  | 低电平 |
| `dri_ll_gpio_level_t` | `GPIO_LEVEL_HIGH` | 高电平 |

#### GPIO 模式类型

| 类型                  | 取值                                  | 说明         |
| --------------------- | ------------------------------------- | ------------ |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_ANALOG`             | 模拟输入     |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_INPUT_FLOATING`     | 浮空输入     |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_INPUT_PULL_UP`      | 上拉输入     |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_INPUT_PULL_DOWN`    | 下拉输入     |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_OUTPUT_PP`          | 通用推挽输出 |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_OUTPUT_OD`          | 通用开漏输出 |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_AF_PP`              | 复用推挽输出 |
| `dri_ll_gpio_mode_t`  | `GPIO_MODE_AF_OD`              | 复用开漏输出 |

#### GPIO 输出速度类型

| 类型                   | 取值                       | 说明           |
| ---------------------- | -------------------------- | -------------- |
| `dri_ll_gpio_speed_t`  | `GPIO_SPEED_NONE`   | 不使用输出速度 |
| `dri_ll_gpio_speed_t`  | `GPIO_SPEED_2MHZ`   | 2MHz           |
| `dri_ll_gpio_speed_t`  | `GPIO_SPEED_10MHZ`  | 10MHz          |
| `dri_ll_gpio_speed_t`  | `GPIO_SPEED_50MHZ`  | 50MHz          |

#### 初始化结构体

```c
typedef struct
{
    dri_ll_gpio_port_t port;
    dri_ll_gpio_pin_t pin;
    dri_ll_gpio_mode_t mode;
    dri_ll_gpio_speed_t speed;
    dri_ll_gpio_level_t initial_level;
} dri_ll_gpio_init_t;
```

字段说明：

| 字段            | 说明                                                        |
| --------------- | ----------------------------------------------------------- |
| `port`          | 目标 GPIO 端口                                              |
| `pin`           | 目标 GPIO 引脚                                              |
| `mode`          | GPIO 工作模式，直接描述输入类型或输出类型                   |
| `speed`         | 输出速度，仅输出模式下有效；输入模式下会被忽略              |
| `initial_level` | 输出初始电平，仅输出模式下有效                              |

#### 用户接口列表

| 接口                                      | 返回值                 | 说明                                           |
| ----------------------------------------- | ---------------------- | ---------------------------------------------- |
| `dri_ll_gpio_init(&config)`               | `void`                 | 按结构体配置初始化一个引脚                     |
| `dri_ll_gpio_write_pin(port, pin, level)` | `void`                 | 写单个引脚输出电平                             |
| `dri_ll_gpio_set_pin(port, pin)`          | `void`                 | 单个引脚置高                                   |
| `dri_ll_gpio_reset_pin(port, pin)`        | `void`                 | 单个引脚置低                                   |
| `dri_ll_gpio_toggle_pin(port, pin)`       | `void`                 | 翻转单个引脚输出电平                           |
| `dri_ll_gpio_read_input_pin(port, pin)`   | `dri_ll_gpio_level_t`  | 返回输入电平，非法引脚返回 `GPIO_LEVEL_LOW`    |
| `dri_ll_gpio_read_output_pin(port, pin)`  | `dri_ll_gpio_level_t`  | 返回输出电平，非法引脚返回 `GPIO_LEVEL_LOW`    |
| `dri_ll_gpio_read_input_port(port)`       | `uint16_t`             | 返回整个端口的输入值                           |
| `dri_ll_gpio_read_output_port(port)`      | `uint16_t`             | 返回整个端口的输出值                           |
| `dri_ll_gpio_write_port(port, value)`     | `void`                 | 直接写整个端口输出值                           |

## 4. 接口行为说明

### 4.1 `dri_ll_gpio_init`

功能：

- 先根据 `mode` / `speed` 组合编码出 STM32F1 的 4bit `cfg_bits`
- 如果是上拉或下拉输入模式，会先根据输入类型设置 `ODR` 对应位
- 如果是输出模式，会先根据 `initial_level` 预装载输出值
- 最后写入 `CRL/CRH` 完成配置

说明：

- `GPIO_MODE_INPUT_PULL_UP` / `GPIO_MODE_INPUT_PULL_DOWN` 会编码成输入上拉/下拉配置
- `speed` 只在输出模式下参与编码，输入模式下会被忽略
- `initial_level` 只在输出模式下生效
- 用户层不再公开保留配置位型；基础层 raw 接口仍会拒绝把保留值 `0xC` 写入寄存器
- 调用本接口前，必须先由外部 `RCC` 模块打开对应 GPIO 端口时钟
- 接口内部不返回错误码，参数非法时直接返回

### 4.2 `dri_ll_gpio_write_mask_raw`

功能：

- 通过 `BSRR` 一次写操作同时完成置位和复位
- 适合需要原子操作的输出控制场景

说明：

- `set_mask` 中为 `1` 的位会被置高
- `reset_mask` 中为 `1` 的位会被置低

### 4.3 `dri_ll_gpio_toggle_pin`

功能：

- 先读取 `ODR`
- 再根据当前位状态决定置位或复位

说明：

- 该接口基于读改写逻辑实现翻转
- 在 Cortex-M 目标上，当前实现会在翻转窗口内短暂进入临界区，降低与中断并发访问时的状态竞争风险

## 5. 使用示例

### 5.1 配置 PC13 为推挽输出并点灯

```c
#include "dri_ll_gpio.h"

int main(void)
{
    /* 先调用你们团队的 RCC 接口打开 GPIOC 时钟 */

    dri_ll_gpio_init_t led = {
        .port = DRI_LL_GPIO_PORT_C,
        .pin = DRI_LL_GPIO_PIN_13,
        .mode = GPIO_MODE_OUTPUT_PP,
        .speed = GPIO_SPEED_2MHZ,
        .initial_level = GPIO_LEVEL_HIGH,
    };

    dri_ll_gpio_init(&led);

    while (1)
    {
        dri_ll_gpio_toggle_pin(DRI_LL_GPIO_PORT_C, DRI_LL_GPIO_PIN_13);
    }
}
```

### 5.2 配置 PA0 为上拉输入并读取按键

```c
#include "dri_ll_gpio.h"

int main(void)
{
    /* 先调用你们团队的 RCC 接口打开 GPIOA 时钟 */

    dri_ll_gpio_init_t key = {
        .port = DRI_LL_GPIO_PORT_A,
        .pin = DRI_LL_GPIO_PIN_0,
        .mode = GPIO_MODE_INPUT_PULL_UP,
        .speed = GPIO_SPEED_NONE,
        .initial_level = GPIO_LEVEL_LOW,
    };

    dri_ll_gpio_level_t level;

    dri_ll_gpio_init(&key);

    while (1)
    {
        level = dri_ll_gpio_read_input_pin(DRI_LL_GPIO_PORT_A, DRI_LL_GPIO_PIN_0);

        if (level == GPIO_LEVEL_LOW)
        {
            dri_ll_gpio_set_pin(DRI_LL_GPIO_PORT_C, DRI_LL_GPIO_PIN_13);
        }
        else
        {
            dri_ll_gpio_reset_pin(DRI_LL_GPIO_PORT_C, DRI_LL_GPIO_PIN_13);
        }
    }
}
```

## 6. 开发注意事项

- 本模块当前只负责 GPIO 功能本身，不负责端口时钟开启。
- 在调用 `dri_ll_gpio_init` 或其他 GPIO 访问接口前，需要先由外部 `RCC` 模块打开对应端口时钟。
- 如果后续要支持外部中断，需要继续补充 `AFIO` 和 `EXTI` 模块。
- 复用功能引脚可以通过 `mode + speed` 表达复用推挽/复用开漏，但相关外设功能仍需由对应外设驱动完成。
- `dri_ll_gpio_write_port` 会直接覆盖整个端口的 `ODR` 值，使用时要注意不要误改其他引脚。
- 用户层使用拆分后的语义枚举；如果需要直接对照 `STM32F1` 的 `CNF[1:0]`/`MODE[1:0]`，请使用基础层 `cfg_bits` 接口。

## 7. 建议扩展方向

- 增加 `dri_ll_rcc` 模块，统一管理外设时钟
- 增加 `dri_ll_afio` 模块，支持引脚重映射
- 增加 `dri_ll_exti` 模块，支持 GPIO 外部中断
- 增加更高一层的 `dri_gpio` 业务接口，屏蔽模式细节
